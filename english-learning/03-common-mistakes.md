# ⚠️ Common Mistakes – Quick Reference

Mistakes found repeatedly in professional writing, with quick fixes.

---

## Spelling Mistakes

| Wrong | Correct |
|-------|---------|
| `dicussed` | `discussed` |
| `compatability` | `compatibility` |
| `colaque` | `colleague` |
| `alos` | `also` |
| `yeterday` | `yesterday` |
| `nurtue` | `nurture` |
| `reqwirte` | `rewrite` |
| `analyzes` | `analyses` (British English, formal) |

---

## Grammar Mistakes

| Wrong | Correct | Rule |
|-------|---------|------|
| `we would require` | `we require` | Don't use "would" for direct requests |
| `we have been mitigated` | `we have mitigated` | have + past participle only |
| `we are in process for planning` | `we can proceed with planning` | Natural English phrasing |
| `on next week` | `next week` | No "on" before "next week" |
| `in an urgent case` | (remove) | Redundant after "Urgent:" label |
| `anomalies (if any)` | `anomalies` | "Any" already implies "if any" |
| `operators components` | `operator components` | Noun modifiers stay singular |
| `N-1- version` | `N-1 version` | No trailing hyphen |

---

## Capitalization Mistakes

| Wrong | Correct |
|-------|---------|
| `openshift` | `OpenShift` |
| `openssl` | `OpenSSL` |
| `redis operator` | `Redis Operator` |
| `V-4-19` | `v4.19` |
| `rhel 9` | `RHEL 9` |

## Tone Mistakes (Too Casual / Weak)

| Casual / Weak | Professional / Strong |
|---------------|----------------------|
| `Please let us know if` | `Should you require` |
| `we are tracking` | `we are closely tracking` |
| `we are checking` | `we are conducting a detailed analysis` |
| `case` | `support case` |
| `fix` | `remediation` |
| `updates` | `progress updates will be communicated` |
| `delays` | `undue delays` |
